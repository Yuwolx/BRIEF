# BRIEF

**업무 요청을 보내기 전에, 한 번 더 정리합니다.**

BRIEF는 사내에서 발생하는 업무 요청을 보다 명확하게 전달할 수 있도록 돕는  
경량화된 내부 커뮤니케이션 도구입니다.  
메시지를 대신 써주는 것이 아니라, 요청의 의도와 맥락을 먼저 정리해  
처음부터 오해 없는 소통이 이루어지도록 설계되었습니다.

---

## 프로젝트 배경

조직 내에서는 PM이 아닌 구성원도 타 부서와 직접 소통해야 하는 경우가 많습니다.  
그러나 서로의 역할과 기준을 충분히 이해하지 못한 상태에서 요청이 오가면서,  
불필요한 반복 설명이나 감정적인 오해가 발생하는 경우를 자주 경험했습니다.

BRIEF는 이러한 문제를 개인의 커뮤니케이션 능력이 아니라,  
**요청이 만들어지는 구조의 문제**로 보고 출발한 프로젝트입니다.

---

## 핵심 기능

- 보내는 사람과 받는 사람의 역할 기반 요청 구조화
- 요청 목적 명확화
- 선택형 맥락 입력 (텍스트 및 파일)
- 자연스러운 업무 요청 메일 초안 생성
- 사용자 지시 기반 수정 요청
- 한글 기본 UI 및 선택형 영어 메일 생성

---

## 제품 철학

BRIEF는 사람을 대신해 소통하는 AI가 아니라,  
**사람이 더 정확하게 요청할 수 있도록 돕는 도구**를 목표로 합니다.

AI는 생각을 대신하지 않고,  
복잡한 정리 과정을 줄여주는 역할에 집중합니다.

---

## 프로젝트 상태

본 프로젝트는 사용자 경험과 제품 방향성 검증을 위한 **MVP 단계**입니다.  
기능과 구현 방식은 지속적으로 개선될 수 있습니다.

---

## 🚀 실행 방법 (로컬 환경)

아래 절차를 통해 BRIEF를 로컬 환경에서 실행할 수 있습니다.

### 1. 저장소 클론

```bash
git clone <repository-url>
cd brief
````

---

### 2. 패키지 설치

본 프로젝트는 **pnpm**을 패키지 매니저로 사용합니다.

```bash
npm install -g pnpm
pnpm install
```

> pnpm이 설치되어 있지 않은 경우 위 명령어로 먼저 설치합니다.

---

### 3. 환경 변수 설정

프로젝트 루트에 `.env.local` 파일을 생성하고 아래 내용을 추가합니다.

```env
OPENAI_API_KEY=your_openai_api_key
```

> OpenAI API 키는 메일 생성 기능에 사용됩니다.
> API 호출이 불가능한 경우에도 UX가 유지되도록 fallback 응답이 구현되어 있습니다.

---

### 4. 개발 서버 실행

```bash
pnpm dev
```

실행 후 브라우저에서 아래 주소로 접속합니다.

```
http://localhost:3000
```

---

## 🧭 주요 사용자 흐름

1. 역할 선택 (보내는 사람 / 받는 사람)
2. 요청 맥락 입력 (선택)
3. 요청 목적 선택
4. AI 추가 질문 응답
5. 업무 요청 메일 초안 생성
6. 사용자 요청에 따른 수정 및 복사

---

## 🛠 기술 스택

* **Framework**: Next.js (App Router)
* **UI**: Tailwind CSS, shadcn/ui
* **State Management**: React Hooks
* **AI Integration**: OpenAI API
* **Package Manager**: pnpm
